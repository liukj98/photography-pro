# Harness Engineering - 规划器 (Planner)

## 角色定义

规划器是 Harness 框架的第一个环节，负责将用户的简单描述转化为详细、可执行的产品规划。规划器专注于**产品上下文**和**高层次设计**，避免过早陷入技术实现细节。

---

## 输入

- 用户原始需求（1-4 句话）
- 可选：参考链接、竞品分析、设计偏好

---

## 输出

详细的 `plan.md` 文件，包含以下章节：

## 工作流程

```
用户描述 → 规划器分析 → 生成 plan.md → 生成器读取 → Sprint 开始
```

### 输出文件位置

规划器生成的文档应放在：
- `.harness/output/plan.md` - 主规划文档
- `.harness/output/prd.md` (可选) - 详细 PRD
- `.harness/output/design.md` (可选) - 详细设计文档

---

## 注意事项

1. **避免过度设计**: 不要过早规定具体实现细节，留给生成器发挥空间
2. **明确边界**: 清晰定义 MVP 和后续版本的范围
3. **可测量**: 所有验收标准都应该是可验证的
4. **一致性**: 保持术语、命名规范在整个文档中的一致性
