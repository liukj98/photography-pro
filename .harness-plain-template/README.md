# Harness Engineering 纯净模板

> 一个让大模型自主完成产品全生命周期的工程规范。无需人工介入，规划→生成→评估→迭代，直到产品达标交付。

---

## 核心机制

```
                         ┌─────────────────────┐
                         │    规划器 (Planner)   │
                         │  输出：产品规划文档     │
                         └──────────┬──────────┘
                                    │ plan.md
                         ┌──────────▼──────────┐
                    ┌───►│    生成器 (Generator)  │◄───┐
                    │    │  输出：可运行的软件     │    │
                    │    └──────────┬──────────┘    │
                    │               │ 交付物          │
                    │    ┌──────────▼──────────┐    │
                    │    │    评估器 (Evaluator)  │    │
                    │    │  输出：评估报告+反馈   │    │
                    │    └──────────┬──────────┘    │
                    │               │                │
                    │          达标？─┼──否──────────┘
                    │               │
                    │              是
                    └────────── 交付 ──────────┘
```

**只有一个触发点**：人类给出产品需求（1-4 句话）。

之后的一切——规划、编码、测试、评估、修复——全部由三个角色自动循环完成，直到产品通过验收。

---

## 三个角色

| 角色 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **规划器** | 理解需求，制定蓝图 | 产品需求描述 | `plan.md` |
| **生成器** | 按蓝图实现产品 | `plan.md` + 评估反馈 | 可运行的软件 + 实现说明 |
| **评估器** | 审查质量，给出反馈 | 蓝图 + 实现说明 + 运行中的软件 | `evaluation.md`（评分+反馈） |

**通信方式**：角色之间通过读写约定的文件进行通信，无直接对话。

---

## 文件约定

```
.harness/
├── plan.md              ← 规划器写，生成器读
├── sprint.md            ← 生成器写，评估器读
├── evaluation.md        ← 评估器写，生成器读
└── state.md             ← 当前运行状态（任一角色可读写）
```

| 文件 | 写入者 | 读取者 | 触发条件 |
|------|--------|--------|----------|
| `plan.md` | 规划器 | 生成器 | 首次启动或需求变更 |
| `sprint.md` | 生成器 | 评估器 | 每轮迭代完成 |
| `evaluation.md` | 评估器 | 生成器 | 每轮评估完成 |
| `state.md` | 当前角色 | 下一个角色 | 每轮结束后更新 |

---

## 运行协议

### 启动

1. 人类将产品需求写入触发文件，或直接作为初始上下文传入
2. 规划器读取需求，生成 `plan.md`
3. 生成器读取 `plan.md`，开始实现

### 循环

```
生成器完成一轮实现
    → 写入 sprint.md（描述本轮完成了什么）
    → 评估器读取 plan.md + sprint.md + 实际代码
    → 评估器写入 evaluation.md（评分 + 具体问题 + 修复建议）
    → 生成器读取 evaluation.md
    → 如果未达标：修复问题，开始下一轮
    → 如果已达标：写入 state.md 标记完成
```

### 终止

当 `state.md` 中的状态变为 `DELIVERED`，循环停止。
