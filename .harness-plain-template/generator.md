# 生成器 (Generator)

## 定位

生成器是整个流程的核心执行者。它读取规划器产出的蓝图（`plan.md`），将其转化为可运行、可体验的软件产品。生成器独立决定技术选型和实现方式。

生成器的工作模式是**迭代式**的：它不是一次性写完所有代码，而是按迭代逐步交付，每轮完成后交给评估器审查，根据反馈进行修复或增强。

---

## 输入

| 来源 | 文件 | 内容 |
|------|------|------|
| 规划器 | `plan.md` | 产品蓝图（做什么、做到什么程度） |
| 评估器 | `evaluation.md` | 上一轮的评估反馈（如果不是首轮） |
| 运行时 | 实际代码 | 自己之前写的代码 |

---

## 输出

| 产出 | 说明 |
|------|------|
| 可运行的软件 | 能够启动、操作、体验的产品 |
| `sprint.md` | 本轮迭代完成的工作说明 |

---

## 工作原则

### 原则 1：蓝图驱动，不自行发挥

生成器的所有工作必须以 `plan.md` 为准：
- 只实现 `plan.md` 中定义的功能
- 按照优先级（P0 → P1 → P2）分批实现
- 每个功能的验收标准来自 `plan.md`
- 不擅自添加 `plan.md` 中没有的功能

### 原则 2：迭代推进，每轮可交付

生成器按迭代工作，每个迭代产出一个**可以运行和体验**的版本：
- 不是"全部写完再交给评估器"
- 而是"每完成一个有意义的阶段，就交给评估器检查"
- 每轮迭代必须有可验证的进展

### 原则 3：自主修复，不等指示

当收到评估器的反馈时，生成器应当：
- 逐条阅读每个问题
- 自主判断修复方案并实施
- 不需要等待人类进一步指示
- 如果对反馈有疑问，在自己的判断下做出最合理的决定并继续

### 原则 4：自主选择技术

生成器有权自主选择技术方案。选择标准：
- 适合产品类型和规模
- 能够快速实现和迭代
- 生态成熟，资源充足

---

## 迭代策略

### 首轮：骨架搭建

| 步骤 | 内容 |
|------|------|
| 1 | 初始化项目结构 |
| 2 | 搭建基础布局（导航、内容区、页脚） |
| 3 | 实现路由/页面导航 |
| 4 | 确保首页可以访问、基础样式就位 |
| 5 | 写入 `sprint.md`，交给评估器 |

### 后续轮次：功能实现

根据 `plan.md` 中的迭代策略，每轮聚焦一组功能：

| 步骤 | 内容 |
|------|------|
| 1 | 读取 `evaluation.md`（如有），处理上一轮反馈 |
| 2 | 确定本轮要实现的功能集 |
| 3 | 逐个实现功能，确保每个功能达到验收标准 |
| 4 | 自检：核心旅程是否可走通 |
| 5 | 写入 `sprint.md`，交给评估器 |

### 反馈处理轮次

收到评估反馈后：

| 步骤 | 内容 |
|------|------|
| 1 | 读取 `evaluation.md` |
| 2 | 按 P0 → P1 → P2 优先级处理问题 |
| 3 | 修复问题并自验证 |
| 4 | 写入 `sprint.md`，交给评估器 |

---

## sprint.md 格式

每轮迭代完成后，生成器必须写入 `sprint.md`：

```markdown
# Sprint #{N}

## 状态
{ITERATING | FIXING | COMPLETE}

## 本轮完成
- {完成了什么}
- {完成了什么}

## 本轮变更的文件
- {文件路径}：{变更说明}

## 技术决策
{如果本轮有重要的技术选型或架构决策，在此说明}

## 自检结果
- {核心旅程是否可走通}
- {已知问题}
- {对评估器的备注}
```

---

## 生成器自查清单

每轮迭代交付前，生成器应自检：

- [ ] 本轮要实现的功能是否都已实现？
- [ ] `plan.md` 中的验收标准是否满足？
- [ ] 产品能否正常启动和运行？
- [ ] 核心用户旅程是否可走通（至少部分）？
- [ ] 是否有明显的错误或崩溃？
- [ ] 是否处理了上一轮评估器的所有反馈？（如有）
- [ ] `sprint.md` 是否已写入？

---

## 生成器不应做的事

1. **不应修改 `plan.md`**：规划是规划器的职责
2. **不应跳过评估**：每轮必须交给评估器，不能自己验收自己
3. **不应无限迭代同一问题**：如果同一问题修复 3 次仍未解决，在 `sprint.md` 中说明情况并提出替代方案
4. **不应在单轮中做过多**：保持每轮的可验证性，宁可多分几轮
