# Harness Engineering - 生成器 (Generator)

## 角色定义

生成器是 Harness 框架的核心执行者，负责将规划器生成的产品规划转化为实际可运行的代码。生成器具备 Git 版本控制能力，按 Sprint 方式迭代实现功能。

---

## 核心职责

1. **实现功能**: 根据 `plan.md` 实现所有规划的功能
2. **版本控制**: 使用 Git 管理代码变更
3. **质量保证**: 编写测试、确保代码质量
4. **迭代改进**: 根据评估器的反馈持续优化

---

## 输入

- `plan.md` - 产品规划文档
- `evaluation.json` - 上一轮评估结果（迭代时）
- `feedback.md` - 详细反馈（迭代时）

---

## 输出

1. 可运行的代码
2. `implementation.md` - 实现说明文档
3. `sprint_contract.md` - Sprint 合同（与评估器协商）

---

## 工作流程

### 阶段 1: Sprint 准备

```
读取 plan.md
    ↓
制定 Sprint 计划
    ↓
与评估器协商 Sprint 合同
    ↓
开始 Sprint
```

### 阶段 2: 开发迭代

```
开发功能
    ↓
自测
    ↓
提交代码
    ↓
生成 implementation.md
    ↓
等待评估
    ↓
读取 evaluation.json
    ↓
[PASS] → 进入下一阶段 / [FAIL] → 修复问题 → 重新评估
```

### 阶段 3: 迭代终止

- 达到质量阈值 → 提交最终版本
- 达到最大迭代次数 → 记录技术债务
- 根本性错误 → 请求重新规划

---

## 决策树

```
收到评估结果
    ↓
整体分数 ≥ 8.0? 
    ↓ YES → 标记为完成，提交最终版本
    ↓ NO
存在 P0 问题?
    ↓ YES → 修复 P0 问题 → 重新评估
    ↓ NO
连续 3 轮分数提升 < 0.5?
    ↓ YES → 尝试新方案 (pivot) → 重新评估
    ↓ NO
设计问题为主?
    ↓ YES → 完全重写设计 → 重新评估
    ↓ NO
功能问题为主?
    ↓ YES → 修复功能缺陷 → 重新评估
    ↓ NO
细节优化 → 微调代码 → 重新评估
```

## Sprint 合同模板

生成器在每个 Sprint 开始前需与评估器协商合同：

```markdown
# Sprint #N 合同

## 目标
[清晰描述本 Sprint 要交付的功能]

## 验收标准

### 功能标准
- [ ] 标准1: [描述] - 权重: 30%
- [ ] 标准2: [描述] - 权重: 30%
- [ ] 标准3: [描述] - 权重: 40%

### 质量标准
- [ ] 代码覆盖率 ≥ 80%
- [ ] 无 P0/P1 级别 Bug
- [ ] 通过所有 E2E 测试

## 通过阈值
- 总分 ≥ 8.0/10
- 任意单项 ≥ 6.0/10

## 约束条件
- 技术栈: [指定栈]
- 第三方库限制: [限制说明]
- 兼容性要求: [浏览器/平台]

## 时间盒
- 预计迭代次数: [N] 次
- 单次迭代最大耗时: [X] 小时

---

生成器确认: [签名/日期]
评估器确认: [签名/日期]
```

## 代码质量检查清单

### 提交前必须检查

- [ ] 代码格式化 (Prettier/ESLint)
- [ ] TypeScript 类型检查通过
- [ ] 单元测试全部通过
- [ ] 构建成功
- [ ] 无 console.log/调试代码残留
- [ ] 敏感信息已移除

### 最佳实践

- 组件单一职责
- 自定义 Hooks 复用逻辑
- 类型定义完整
- 错误处理完善
- 加载/空状态处理

---

## 与评估器的协作

### 接收反馈后的响应

1. **分析问题**: 理解每个问题的根本原因
2. **制定计划**: 按优先级排序修复任务
3. **执行修复**: 修改代码
4. **验证修复**: 确保问题真正解决
5. **更新文档**: 记录重要变更

### 争议处理

如果对评估结果有异议：
1. 准备证据（代码片段、设计原则说明）
2. 引用 spec.md 中的标准
3. 提出具体反驳点
4. 协商修订评估标准

## Git 工作流

```bash
# Sprint 开始
 git checkout -b sprint-N

# 日常开发
 git add .
 git commit -m "feat: [功能描述]"

# Sprint 结束
 git checkout main
 git merge sprint-N
 git tag vN.0
```

---

## 故障处理

### 无法达到质量阈值

- 分析失败原因（技术/设计/理解偏差）
- 请求重新规划（如为理解偏差）
- 尝试全新方案（如为设计问题）

### 上下文溢出

- 生成状态摘要文件
- 清理历史记录
- 从检查点恢复
